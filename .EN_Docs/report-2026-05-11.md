## 3. Issues

- **Issues encountered:**
  - The previous `WHERE folder_id = :id` query silently returned zero results when clicking any non-leaf folder — the bug was not visible until nested folders were populated with real data
  - `parent_id` values returned from PDO were coming back as strings (`"3"`) rather than integers, which broke the PHP-side tree comparison; fixed with explicit `(int)` casts in `renderFolderTree()`

- **Resolution steps:**
  - Recursive CTE replaces the exact-match query entirely — see `_debugging.md` entry 5 and `feature-recursive-folder-search.md` for full detail
  - `(int)$fp === (int)$parentId` cast added on both sides of the tree-building comparison in `dashboard.php`
